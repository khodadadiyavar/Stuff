# Linux for Beginners: Session 4 Answers

## Answer Key

### 1. Find the exact version of the kernel running on your system.

Answer:

Use:

```bash
uname -r
```

Description:

In this question, you learn how to check the exact kernel version currently running on your Linux system.

How to solve:

Run:

```bash
uname -r
```

This prints the exact kernel release for the current system. The value will vary from machine to machine.

### 2. Determine which init process is used by your system.

Answer:

Use:

```bash
ps -p 1
```

Description:

In this question, you learn how to inspect process ID 1 and identify the init process used by your system.

How to solve:

Run:

```bash
ps -p 1
```

Look at the process shown for PID 1. On many modern Linux systems, this will be `systemd`.

### 3. Find the default `systemd` target set on your system.

Answer:

Use:

```bash
systemctl get-default
```

Description:

In this question, you learn how to check which `systemd` target your system is configured to use by default at boot.

How to solve:

Run:

```bash
systemctl get-default
```

This prints the configured default target, such as `graphical.target` or `multi-user.target`.

### 4. Change the default target to `multi-user.target`.

Answer:

Use:

```bash
sudo systemctl set-default multi-user.target
```

Description:

In this question, you learn how to change the default `systemd` target and verify that the change was applied successfully.

How to solve:

Run:

```bash
sudo systemctl set-default multi-user.target
```

Then verify it:

```bash
systemctl get-default
```

The verification command should now print:

```text
multi-user.target
```

### 5. Print the messages generated by the kernel.

Answer:

Use:

```bash
dmesg
```

Description:

In this question, you learn the standard command used to display messages generated by the kernel.

How to solve:

Run:

```bash
dmesg
```

If access is restricted on the system, try:

```bash
sudo dmesg
```

### 6. Find how many block devices of type `disk` are present on your system.

Answer:

Use:

```bash
lsblk
```

Description:

In this question, you learn how to list block devices and count only the entries whose type is `disk`.

How to solve:

Run:

```bash
lsblk
```

Look at the `TYPE` column and count the rows where the value is `disk`.
The final number depends on the machine.

### 7. Determine which directory contains the device files related to the block devices shown by `lsblk`.

Answer:

The directory is:

```text
/dev
```

Description:

In this question, you learn where Linux exposes block devices as device files in the filesystem.

How to solve:

Run:

```bash
lsblk
```

Then inspect:

```bash
ls /dev | head
```

You can also confirm by listing specific block device paths such as `/dev/sda`, `/dev/vda`, or `/dev/nvme0n1`, depending on the system.

### 8. Find the vendor name of the Ethernet controller used by your system.

Answer:

Use:

```bash
lspci
```

Description:

In this question, you learn how to inspect PCI hardware information and identify the vendor of your Ethernet controller.

How to solve:

Run:

```bash
lspci | grep -i ethernet
```

If the command is unavailable, install `pciutils` first using the package manager.

The vendor name is part of the output line, for example `Intel Corporation`, `Broadcom Inc.`, or another vendor depending on the machine.

### 9. Identify the recommended directory for installing a third-party IDE system-wide.

Answer:

The recommended directory is:

```text
/opt
```

Description:

In this question, you learn the standard Linux location used for optional third-party software installed system-wide.

How to solve:

Inspect the top-level directories:

```bash
ls /
```

If needed, read the manual page describing filesystem hierarchy conventions:

```bash
man hier
```

`/opt` is the standard location for optional add-on software such as third-party IDEs installed system-wide.
