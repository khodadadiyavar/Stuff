# Linux for Beginners: Session 4 Homework

## Objective

Practice working with:

- kernel information
- system initialization
- `systemd` targets
- kernel log messages
- block devices
- hardware inspection
- standard Linux filesystem layout

## Important Notes

- Run the commands on your own Linux system and record the output you see.
- Some answers depend on the system you are using, so do not guess.
- A few commands may require `sudo` on your machine.
- If a command shows too much output, use tools such as `head`, `grep`, or `less` to inspect it more easily.

## Questions

### 1. Find the exact version of the kernel running on your system.

Run the correct command and record the full kernel version shown on your machine.

### 2. Determine which init process is used by your system.

Run the correct command and record the name of the init system in use.

### 3. Find the default `systemd` target set on your system.

Run the correct command and record the current default target.

### 4. Change the default target to `multi-user.target`.

Run the correct command to make the change. Then verify that the default target is now `multi-user.target`.

### 5. Print the messages generated by the kernel.

Run the correct command that displays kernel messages.

### 6. Find how many block devices of type `disk` are present on your system.

Run the correct command and record the number you get on your machine.

### 7. Determine which directory contains the device files related to the block devices shown by `lsblk`.

Run the correct command to inspect the system and identify the directory.

### 8. Find the vendor name of the Ethernet controller used by your system.

Run the correct command and record the vendor shown on your machine.

### 9. Identify the recommended directory for installing a third-party IDE system-wide.

Inspect the system layout and determine which directory is the best fit for optional third-party software installed outside the package manager.

## Submission Checklist

Before you finish, confirm that:

- you ran commands on your own system for every question
- you used `sudo` where required
- you verified question 4 after changing the default target
- you wrote down system-specific outputs for questions 1, 3, 6, and 8
